# 登録/編集フローと運用（ドラフト）

## 1. 前提
- 当面は開発者本人が手動で更新
- 登録用JSONは `register/` に配置（ID未指定可、参照はnameでも可）
- 確定データは `data/` に保存（UUID参照のみ）
- CLIでUUID採番とname→UUID解決、整合性チェックを実施

## 2. 登録/編集フロー
- 入力（下書き）：`register/` にJSONを作成し、`group_id` や `attendees` などの参照はnameで記述可
- CLI実行：登録用JSONを読み込み、UUID採番とname解決を行い、フォーマット/必須チェックを実施
- 確定データ生成：UUID参照のみのJSONを `data/` に出力
- 確認：生成物を目視確認し、必要に応じて修正→再実行
- 公開反映：`data/` の更新を反映（後続のデプロイ/公開手順に従う）
- 変更履歴：差分管理とロールバック手段、誰がいつ更新したかの記録方法を定義

## 3. 運用ルール
- 更新トリガー：会議情報公開のたびに手動更新
- 更新フロー：下書き→CLI生成→確認→反映 の基本手順
- 重複・修正：既存会議の上書き条件、誤登録時の修正手順をここに記す

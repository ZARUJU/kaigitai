# GOAL
- 閲覧専用サイトを用意し、データを静的化して一般公開できる状態にする
- Frozen-Flaskで静的生成し、GitHub Pages等にデプロイ可能にする

# ユーザーからの依頼
- 新たに viewer.py を作成し、CRUD の R だけを提供する閲覧版を用意する
- Frozen-Flask を利用して静的サイトを作り、GitHub Pages で公開できるようにする
- 編集/追加系のUIを廃し、全ページで「データの正確性は保証しない」旨の文言を表示する
- まずはタスク作成と閲覧サイトの仕様策定

# タスク
## 現状
- Flask管理UIはCRUD想定で、編集・追加ボタンが存在する
- 静的生成や閲覧専用向けのエントリポイントは無い
- データは `data/` 配下のJSONで管理し、templatesは管理UI向けレイアウト
## 変更後
- 閲覧専用エントリポイント `viewer.py` を追加（Rのみ）
- Frozen-Flaskで全公開ページを静的化するビルドフローを用意
- 全ページに免責文言を表示し、編集・追加ボタンは非表示/削除
- GitHub Pages 等で公開できる成果物（`build/` など）を生成できる状態
## 手順
- 閲覧サイトの仕様策定（ページ/ルーティング/表示要素/免責文言/ナビ）
- `viewer.py` で読み取り専用のFlaskアプリを実装（POST禁止、編集リンクなし、ルーティング一覧が見やすい構成）
- Frozen-Flask設定を追加し、`build/` に全ルートをプリレンダリングするスクリプトを整備
- テンプレートはコンポーネント/フラグで共用し、閲覧用では免責表示・ボタン非表示にする
- GitHub Pages公開手順をドキュメント化（ビルドコマンド、出力先ディレクトリ）
